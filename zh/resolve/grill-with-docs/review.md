# 技能赏析：grill-with-docs

**分类**：engineering | **技能名**：grill-with-docs

---

## 技能定位

`grill-with-docs` 是 `grill-me` 的进化版本——在保留"逐一追问、压力测试设计方案"核心机制的基础上，叠加了**领域文档感知**能力。它不仅是一个苏格拉底式的设计审查员，更是一个能读懂项目语言体系（CONTEXT.md）和历史决策（ADR）的"有记忆的审讯者"。

适用场景：当你想把一个新计划放到项目已有的领域模型和术语体系下进行压力测试，同时希望在对话过程中同步更新文档时。

---

## 核心设计哲学

### 1. 文档即活体，而非归档

大多数项目的文档是"写完就放着"的——它们记录过去，但不参与当下的决策。`grill-with-docs` 打破了这一惯例：**在对话过程中实时更新 CONTEXT.md**，而不是等会话结束后再整理。

> "When a term is resolved, update CONTEXT.md right there. Don't batch these up — capture them as they happen."

这背后的哲学是：**决策的价值在于被捕获的那一刻**。延迟记录会导致细节流失、语境丢失。实时写入让文档成为对话的副产品，而非额外的负担。

### 2. 术语即契约

CONTEXT.md 的设计不是词汇表，而是**团队的语言契约**。每个术语都有精确定义、明确的"避免使用"别名，以及与其他术语的关系描述。

`grill-with-docs` 在审查过程中会主动对照这份契约：当用户使用了模糊词汇或与已有定义冲突的术语时，立即叫停并要求澄清。这将"术语漂移"扼杀在萌芽状态——而术语漂移正是大型项目中沟通失效的主要根源之一。

### 3. ADR 的克制哲学

技能对 ADR（架构决策记录）的态度极为克制，设置了三重门槛：
- **难以逆转**（reversibility）
- **脱离上下文会令人困惑**（surprising without context）
- **真实的权衡取舍**（real trade-off）

三者缺一不可才建议创建 ADR。这是对"文档膨胀"的主动抵抗——只记录真正值得记录的决策，避免 ADR 目录变成无人阅读的档案馆。

---

## 关键机制细节分析

### 文件结构的惰性创建（Lazy Creation）

技能不要求项目预先建好文档结构。CONTEXT.md 在"第一个术语被解析时"才创建，`docs/adr/` 在"第一个 ADR 被需要时"才创建。这降低了使用门槛，避免了"先搭脚手架再干活"的仪式感。

### 单/多上下文的自动推断

技能能自动识别仓库是单一领域上下文还是多上下文结构：
- 存在 `CONTEXT-MAP.md` → 多上下文，读取地图找到各子上下文
- 只有根目录 `CONTEXT.md` → 单上下文
- 两者都没有 → 惰性创建

这种自适应让技能能无缝应用于从小型单体到大型微服务的各类项目。

### 代码与文档的交叉验证

当用户描述某个行为时，技能会主动去检查代码是否与描述一致。如果发现矛盾，立即暴露：

> "Your code cancels entire Orders, but you just said partial cancellation is possible — which is right?"

这是一个极其有价值的机制：**文档和代码的分歧往往是 bug 的温床**，而这个技能把发现分歧变成了对话的一部分。

### 场景压力测试

对于领域关系的讨论，技能会主动构造边界场景来逼迫用户精确化：

> "No — an Invoice is only generated once a Fulfillment is confirmed."

这种对话示例内嵌在 CONTEXT-FORMAT.md 中，既是格式规范，也是思维方式的示范——用具体场景代替抽象定义。

---

## 亮点总结

| 亮点 | 说明 |
|------|------|
| **实时文档更新** | 决策即时落地，不依赖事后整理 |
| **术语守门人** | 主动识别并纠正术语漂移 |
| **ADR 三重门槛** | 克制地记录真正值得记录的决策 |
| **惰性文件创建** | 零摩擦接入，无需预建文档结构 |
| **代码-文档交叉验证** | 暴露描述与实现之间的隐性矛盾 |
| **自适应上下文结构** | 单/多上下文仓库均可无缝使用 |

`grill-with-docs` 的精妙之处在于：它把"写文档"这件通常被视为负担的事，变成了设计对话的自然延伸。文档不再是会议结束后的作业，而是思考过程本身的结晶。
