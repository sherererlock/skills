# 深度模块

摘自《软件设计哲学》（A Philosophy of Software Design）：

**深度模块** = 小接口 + 大量实现

```text
┌─────────────────────┐
│       小接口        │  ← 方法少，参数简单
├─────────────────────┤
│                     │
│                     │
│      深度实现       │  ← 隐藏复杂的逻辑
│                     │
│                     │
└─────────────────────┘
```

**浅层模块** = 大接口 + 少量实现（应避免）

```text
┌─────────────────────────────────┐
│             大接口              │  ← 方法多，参数复杂
├─────────────────────────────────┤
│           单薄的实现            │  ← 仅仅是透传
└─────────────────────────────────┘
```

在设计接口时，请思考：

- 我能否减少方法的数量？
- 我能否简化参数？
- 我能否在内部隐藏更多的复杂性？
